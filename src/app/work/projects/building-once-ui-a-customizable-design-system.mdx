## Overview

本プロジェクトは、私自身の学習と実践を長期的に統合していく
**メインプロジェクト**として設計したパーソナル Web プラットフォームです。

ポートフォリオおよびブログ機能を基盤とし、
将来的には **EC（オンラインショップ）機能の実装**を行うことで、
フロントエンドからバックエンドまでを一貫して開発する
フルスタック構成を目指しています。

---

## Key Features

- **MDX ベースのコンテンツ管理**
`.mdx` ファイルによるブログ・記事管理を採用し、
データベースに依存しないシンプルな運用を実現しています。

- **ポートフォリオ / プロジェクト管理機能**
制作物や学習成果を整理し、分かりやすく発信できる構成にしています。

- **拡張性を重視した設計**
将来的な EC 機能追加を前提とし、
コンポーネント設計やディレクトリ構成を整理しています。

- **写真・ビジュアルコンテンツ対応**
写真やビジュアル要素を活かした表現が可能なレイアウトを実装しています。

---

## Technologies Used

- **Next.js**
モダンなフロントエンド開発のための React フレームワーク。

- **TypeScript**
型安全性を確保し、保守性の高いコードを書くために使用。

- **MDX**
Markdown と React を組み合わせた柔軟なコンテンツ管理。

- **Java / Spring Boot（予定）**
バックエンド API およびビジネスロジック実装のために導入予定。

- **RDB（予定）**
商品・ユーザー・注文情報を管理するためのデータベース設計を想定。

---

## Challenges and Learnings

Next.js における App Router やファイルベースルーティング、
MDX を用いたコンテンツ管理の理解が、最初の大きな課題でした。

試行錯誤を通じて、
プロジェクト構成やディレクトリ設計が開発効率や拡張性に
大きく影響することを学びました。
また、機能性とデザインのバランスについても意識するようになりました。

---

## Roadmap（今後の開発予定）

- **Phase 1**：ブログ・ポートフォリオ機能の安定化
- **Phase 2**：商品一覧・商品詳細ページの実装
- **Phase 3**：Spring Boot によるバックエンド API 構築
- **Phase 4**：ユーザー管理・注文管理機能の追加
- **Phase 5**：決済処理を含む EC 機能の完成（学習目的）

---

## Outcome
